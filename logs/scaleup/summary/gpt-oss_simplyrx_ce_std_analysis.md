# gpt-oss simplyrx: CE vs STD

分析目录：`logs/scaleup/icl_gen_simplyrx/model=gpt-oss`

说明：
- 单元格格式：`solved/total [progress]; s=<med #solvedsamples>; r=<med #solved rounds>`
- `s`：只在 solved runs 上统计，取首次解出时 `NumTrainingSamples` 的中位数
- `r`：只在 solved runs 上统计，取首次解出轮次（`epoch-*` 下首次 `Accuracy=1` 的轮次）的中位数

## 总体观察

- `ce/agentic_reflection` 整体最强，覆盖面和样本效率都明显优于 `std`，尤其在 `StarDepth=1/2/3` 上提升最明显。
- `ce/legacy_best_eval` 明显弱于 `agentic_reflection`，但在少数格子上仍优于 `std`，例如 `states=6, depth=2`。
- `std` 基本只在低复杂度格子稳定；一旦 star depth 增大，很多 cell 迅速掉到 `0/3`。
- 三组结果都在 `StarDepth=4` 上明显变难，但 `agentic_reflection` 仍保留了少量可解格子，而 `std` 和 `legacy_best_eval` 基本接近失效。
