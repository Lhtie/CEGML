# gpt-oss simplyrx: CE vs STD

分析目录：`logs/scaleup/icl_gen_simplyrx/model=gpt-oss`

说明：
- 单元格格式：`solved/total [progress]; s=<med #solved samples>; r=<med #solved rounds>`
- `s`：只在 solved runs 上统计，取首次解出时 `NumTrainingSamples` 的中位数
- `r`：只在 solved runs 上统计，取首次解出轮次的中位数，按 1-based 轮次计数
- 当前 `CE / agentic_reflection` 日志只覆盖 `89` 个 runs，因此它的总体分母和部分 cell 分母会小于 `105`

## 总体观察

- `CE / agentic_reflection` 当前 solve rate 最高，为 `0.831 = 74/89`；但它基于当前可用的 `89` 个 runs，而不是完整 `105` 个 runs。
- 在覆盖完整 `105` 个 runs 的方法里，`CE / single_inference` 最强，solve rate = `0.657 = 69/105`；`STD` 最弱，solve rate = `0.410 = 43/105`。
- 从 states x stardepth 的完整 cell 数看，`CE / agentic_reflection` 为 `22/35`，`CE / single_inference` 为 `20/35`，`STD` 为 `12/35`。
- 三组结果的共同趋势仍然一致：`StarDepth=0` 基本稳定，复杂度升高后覆盖率快速下降；但当前 `agentic_reflection` 在更多中高难度格子上维持住了解题率。
