The regex must be **exact**, **concise**, and obey all size / nesting limits described in sections 3‑5.

---

## 2. ALPHABET & LITERAL RULES
* Fixed alphabet: `Σ = [A‑Za‑z0‑9#]` (uppercase letters, lowercase letters, digits, `#`).  
* Only characters from Σ may appear literally in the regex.  
* **Forbidden constructs**: back‑slashes (`\`), dot (`.`), negated character classes (`[^…]`), anchors (`^`, `$`), word‑boundaries, look‑arounds, back‑references, or any non‑regular construct.

---

## 3. EXTENDED REGEX SYNTAX (WHAT YOU MAY USE)

| Construct | Syntax | Meaning |
|-----------|--------|---------|
| **Literal** | any single symbol from Σ | matches that symbol |
| **Character class** | `[ … ]` | may contain ranges (`A‑Z`, `a‑z`, `0‑9`) and individual literals (including `#`). |
| **Concatenation** | adjacency (`ab`) | “a” followed by “b”. |
| **Union (OR)** | `a|b` | either “a” **or** “b”. |
| **Grouping** | `( … )` | defines scope and precedence. |
| **Intersection (AND)** | `R1 & R2` | language = L(R1) ∩ L(R2). |
| **Complement (NOT)** | `~( R )` | language = Σ* \ L(R). Operand **must** be parenthesised. |
| **Quantifiers** (apply to the immediately preceding atom or group) | `*` (0+), `+` (1+), `?` (0‑1), `{n}`, `{n,m}`, `{n,}` | as usual. |
| **Precedence (high → low)** | Quantifiers → `~` → Concatenation → `&` → `|` | Left‑associative for concatenation, `&`, `|`. |

*All quantifiers count as a single nesting level; total nesting depth of `*`, `+`, `?`, `{}` must not exceed **3**.*

## 5. INFERENCE STRATEGY (STEP‑BY‑STEP)

1. **Identify mandatory / forbidden symbols**  
   * Look for characters (or whole classes) that appear in **every** positive example and **never** in a negative example.  
   * If the rule is “no *X*”, treat *X* as a **forbidden** set (e.g., vowels).  

2. **Express “any allowed symbol”**  
   * The universal set is `[A-Za-z0-9#]`.  
   * To exclude a forbidden set **F**, use **intersection with complement**:  
     ```
     [A-Za-z0-9#] & ~(F)
     ```
     where `F` is a character class containing the forbidden symbols (e.g., `[AEIOUaeiou]`).  
   * **Never** use a negated character class (`[^…]`) – it is prohibited.

3. **Determine length constraints**  
   * If all positives have the same length *k*, use the quantifier `{k}` applied to the “any‑allowed‑symbol” atom.  
   * Example for length 5: `([A-Za-z0-9#] & ~( [AEIOUaeiou] )){5}`.

4. **Validate mentally**  
   * Verify that every positive string satisfies the mandatory/forbidden conditions **and** the length constraint.  
   * Verify that every negative string violates at least one of them.  
   * Pay special attention to edge cases such as strings that contain only lowercase vowels (e.g., “orrrr”). They must be rejected if vowels are forbidden.

5. **Compress the expression**  
   * Remove superfluous parentheses; keep them only where required for precedence or as the operand of `~`.  
   * Use `{n}` instead of repeating the atom *n* times.  
   * Ensure the final length (ignoring whitespace) ≤ 50.

6. **Final sanity checks**  
   * No prohibited symbols appear.  
   * Regex length ≤ 50.  
   * Star‑nesting depth ≤ 3.  
   * All parentheses are balanced; every `~` has a parenthesised operand.
